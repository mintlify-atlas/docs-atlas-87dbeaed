## Prerequisites

Before setting up the VitaFem Backend API, ensure you have the following installed:

- PHP 8.2 or higher
- Composer
- Docker and Docker Compose (for containerized setup)
- MySQL 8.0 or Oracle Database
- Git

## Installation Methods

You can set up the VitaFem Backend API using either Docker (recommended) or a local installation.

### Docker Setup (Recommended)

<Steps>
  <Step title="Clone the repository">
    Clone the VitaFem Backend repository to your local machine:
    ```bash
    git clone <repository-url>
    cd vitafem-backend
    ```
  </Step>

  <Step title="Configure environment variables">
    Copy the example environment file and configure your settings:
    ```bash
    cp .env.example .env
    ```
    
    Update the `.env` file with your database credentials:
    ```env
    APP_NAME=VitaFem
    APP_ENV=local
    APP_KEY=
    APP_DEBUG=true
    APP_URL=http://localhost:8000

    DB_CONNECTION=mysql
    DB_HOST=db
    DB_PORT=3306
    DB_DATABASE=laravel
    DB_USERNAME=laravel
    DB_PASSWORD=laravel
    ```
  </Step>

  <Step title="Build and start Docker containers">
    Use Docker Compose to build and start all services:
    ```bash
    docker-compose up -d --build
    ```
    
    This will start:
    - **Backend**: Laravel API (port 8000)
    - **Database**: MySQL 8.0 (port 3307)
    - **Frontend**: React application (port 3000)
    - **Superset**: Apache Superset for analytics (port 8088)
  </Step>

  <Step title="Generate application key">
    Generate a Laravel application key:
    ```bash
    docker exec -it laravel_app php artisan key:generate
    ```
  </Step>

  <Step title="Run database migrations">
    Execute migrations to set up the database schema:
    ```bash
    docker exec -it laravel_app php artisan migrate
    ```
  </Step>

  <Step title="Seed the database (optional)">
    Populate the database with sample data:
    ```bash
    docker exec -it laravel_app php artisan db:seed
    ```
  </Step>

  <Step title="Verify installation">
    Access the API at:
    ```
    http://localhost:8000
    ```
    
    Check the health endpoint:
    ```bash
    curl http://localhost:8000/api/health
    ```
  </Step>
</Steps>

### Local Setup (Without Docker)

<Steps>
  <Step title="Clone the repository">
    ```bash
    git clone <repository-url>
    cd vitafem-backend
    ```
  </Step>

  <Step title="Install PHP dependencies">
    Install all required PHP packages using Composer:
    ```bash
    composer install
    ```
    
    <Note>
      If you encounter platform requirement issues, use:
      ```bash
      composer install --ignore-platform-reqs
      ```
    </Note>
  </Step>

  <Step title="Configure environment">
    Copy and configure the environment file:
    ```bash
    cp .env.example .env
    ```
    
    Update database settings for local MySQL:
    ```env
    DB_CONNECTION=mysql
    DB_HOST=127.0.0.1
    DB_PORT=3306
    DB_DATABASE=vitafem
    DB_USERNAME=root
    DB_PASSWORD=your_password
    ```
  </Step>

  <Step title="Generate application key">
    ```bash
    php artisan key:generate
    ```
  </Step>

  <Step title="Set up storage permissions">
    Create storage directories and set proper permissions:
    ```bash
    mkdir -p storage/framework/{sessions,views,cache}
    mkdir -p bootstrap/cache
    chmod -R 775 storage bootstrap/cache
    ```
  </Step>

  <Step title="Run migrations">
    ```bash
    php artisan migrate
    ```
  </Step>

  <Step title="Start the development server">
    ```bash
    php artisan serve --host=0.0.0.0 --port=8000
    ```
  </Step>
</Steps>

## Key Dependencies

The VitaFem Backend API includes the following major dependencies:

### Core Framework
- **Laravel Framework**: ^8.75
- **PHP**: ^8.2

### Database
- **yajra/laravel-oci8**: Oracle database driver for Laravel
- **Laravel Sanctum**: ^2.11 - API authentication

### Firebase Integration
- **kreait/firebase-php**: ^6.9
- **kreait/laravel-firebase**: ^4.2
- **firebase/php-jwt**: JWT token handling

### Additional Features
- **Laravel Socialite**: ^5.23 - OAuth authentication
- **fruitcake/laravel-cors**: ^3.0 - CORS support
- **guzzlehttp/guzzle**: ^7.0 - HTTP client

## Environment Variables

### Application Settings
```env
APP_NAME=VitaFem
APP_ENV=production
APP_KEY=base64:your-generated-key
APP_DEBUG=false
APP_URL=https://your-domain.com
```

### Database Configuration
```env
# MySQL
DB_CONNECTION=mysql
DB_HOST=127.0.0.1
DB_PORT=3306
DB_DATABASE=vitafem
DB_USERNAME=your_username
DB_PASSWORD=your_password

# Oracle (see Oracle Configuration guide)
DB_CONNECTION=oracle
DB_HOST=your-oracle-host
DB_PORT=1521
DB_DATABASE=your_service_name
DB_USERNAME=your_username
DB_PASSWORD=your_password
```

### Logging & Debugging
```env
LOG_CHANNEL=stack
LOG_DEPRECATIONS_CHANNEL=null
LOG_LEVEL=debug
```

### Cache & Session
```env
CACHE_DRIVER=file
FILESYSTEM_DRIVER=local
QUEUE_CONNECTION=sync
SESSION_DRIVER=file
SESSION_LIFETIME=120
```

### Mail Configuration
```env
MAIL_MAILER=smtp
MAIL_HOST=mailhog
MAIL_PORT=1025
MAIL_USERNAME=null
MAIL_PASSWORD=null
MAIL_ENCRYPTION=null
MAIL_FROM_ADDRESS=noreply@vitafem.com
MAIL_FROM_NAME="${APP_NAME}"
```

### Firebase Configuration
```env
FIREBASE_CREDENTIALS=/path/to/firebase-credentials.json
FIREBASE_PROJECT=app
FIREBASE_DATABASE_URL=https://your-project.firebaseio.com
```

### Redis (Optional)
```env
REDIS_HOST=127.0.0.1
REDIS_PASSWORD=null
REDIS_PORT=6379
```

### AWS (Optional)
```env
AWS_ACCESS_KEY_ID=
AWS_SECRET_ACCESS_KEY=
AWS_DEFAULT_REGION=us-east-1
AWS_BUCKET=
AWS_USE_PATH_STYLE_ENDPOINT=false
```

## Firebase Setup

<Steps>
  <Step title="Create Firebase project">
    Go to the [Firebase Console](https://console.firebase.google.com/) and create a new project.
  </Step>

  <Step title="Generate service account">
    1. Navigate to Project Settings > Service Accounts
    2. Click "Generate New Private Key"
    3. Save the JSON file securely
  </Step>

  <Step title="Configure credentials">
    Set the path to your Firebase credentials file:
    ```env
    FIREBASE_CREDENTIALS=/path/to/firebase-credentials.json
    ```
    
    Or set the Google application credentials environment variable:
    ```env
    GOOGLE_APPLICATION_CREDENTIALS=/path/to/firebase-credentials.json
    ```
  </Step>

  <Step title="Set Firebase database URL">
    ```env
    FIREBASE_DATABASE_URL=https://your-project-id.firebaseio.com
    ```
  </Step>
</Steps>

## CORS Configuration

The API supports CORS for the following origins by default:

- `https://vetpetfront.onrender.com` (Production)
- `http://localhost:5173` (Development)
- `https://4169f60d.us1a.app.preset.io` (Superset)

To add additional origins, update `config/cors.php`:

```php
'allowed_origins' => [
    'https://your-frontend-domain.com',
    'http://localhost:5173',
],
```

## Troubleshooting

### Storage Permission Issues
If you encounter permission errors:
```bash
chmod -R 775 storage bootstrap/cache
chown -R www-data:www-data storage bootstrap/cache
```

### Composer Memory Limit
If Composer runs out of memory:
```bash
COMPOSER_MEMORY_LIMIT=-1 composer install
```

### Docker Container Not Starting
Check container logs:
```bash
docker logs laravel_app
```

### Database Connection Failed
Verify database credentials and ensure the database service is running:
```bash
docker ps
php artisan config:clear
php artisan cache:clear
```
