## Overview

This endpoint creates a new doctor account, including:
- User credentials for login
- Medical professional information (specialty, license number)
- Default weekly schedule (Monday-Friday, 9 AM - 5 PM)
- Automatic role assignment

<Warning>
  This endpoint requires admin authentication. Include a valid Sanctum token in the Authorization header.
</Warning>

## Request Body

<ParamField body="nombre" type="string" required>
  Full name of the doctor (will be displayed in patient-facing interfaces)
</ParamField>

<ParamField body="correo" type="string" required>
  Email address for login credentials. Must be unique in the system.
</ParamField>

<ParamField body="password" type="string" required>
  Login password. Minimum 6 characters. Will be encrypted using bcrypt.
</ParamField>

<ParamField body="especialidad" type="string" required>
  Medical specialty (e.g., "Ginecología", "Obstetricia", "Medicina General", "Nutrición")
</ParamField>

<ParamField body="cedula" type="string" required>
  Professional license number (cédula profesional) for verification purposes
</ParamField>

## What Happens Behind the Scenes

When you create a doctor, the system automatically:

1. **Creates a User Account** in the USUARIOS table with encrypted password
2. **Assigns Medical Role** (ID_ROLES = 2) in the USUARIO_ROL table
3. **Creates Doctor Profile** in the MEDICO table with specialty and license info
4. **Generates Default Schedule** - Monday through Friday, 9:00 AM to 5:00 PM, with 60-minute appointment slots
5. **Sets Auto-Generated Bio** - "Especialista en [specialty] de VitaFem."

## Error Handling

The endpoint uses database transactions to ensure data consistency. If any step fails, all changes are rolled back:

```javascript
try {
  const result = await createDoctor(doctorData);
  console.log('Doctor created successfully!');
} catch (error) {
  if (error.message.includes('Duplicate entry')) {
    alert('This email is already registered');
  } else {
    console.error('Failed to create doctor:', error);
  }
}
```

## Notes

- Requires authentication with admin privileges
- Email addresses must be unique across all users
- Password is automatically hashed using Laravel's bcrypt
- The system uses database transactions - if creation fails, no partial data is saved
- Doctors can immediately log in and customize their schedule after creation
