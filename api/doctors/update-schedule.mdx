## Overview

This endpoint allows doctors to customize their weekly working schedule, including which days they work, working hours, and appointment duration. The system replaces the entire schedule on update to prevent conflicts.

<Warning>
  This endpoint completely replaces the doctor's existing schedule. Only days marked as `activo: true` will be saved. Make sure to include all working days in your request.
</Warning>

## Path Parameters

<ParamField path="id_usuario" type="integer" required>
  The user ID of the doctor whose schedule is being updated
</ParamField>

## Request Body

Send an array of schedule objects. Only include days where `activo: true` - inactive days will be automatically deleted from the schedule.

<ParamField body="dia_semana" type="integer" required>
  ISO day of week (1-7, where 1 = Monday, 7 = Sunday)
</ParamField>

<ParamField body="activo" type="boolean" required>
  Whether the doctor works on this day. Set to `false` for days off (these can be omitted from request).
</ParamField>

<ParamField body="hora_inicio" type="string" required>
  Start time in 24-hour format (HH:MM). Example: "08:00", "09:30"
</ParamField>

<ParamField body="hora_fin" type="string" required>
  End time in 24-hour format (HH:MM). Must be later than hora_inicio. Example: "17:00", "18:30"
</ParamField>

<ParamField body="duracion_cita" type="integer" required>
  Appointment slot duration in minutes. Common values: 30, 45, 60
</ParamField>

## How It Works

The update process uses a database transaction to ensure data integrity:

1. **Validates Doctor** - Confirms the user is a registered doctor
2. **Begins Transaction** - Ensures all-or-nothing update
3. **Deletes Old Schedule** - Removes all existing schedule entries
4. **Inserts Active Days** - Creates new entries only for days where `activo: true`
5. **Commits Changes** - Saves everything or rolls back on error

<Info>
  Only days with `activo: true` are saved to the database. Days with `activo: false` or omitted entirely result in no appointments being available.
</Info>

## Building a Schedule Editor

```javascript
const ScheduleEditor = ({ doctorUserId }) => {
  const [schedule, setSchedule] = useState([]);
  
  useEffect(() => {
    // Load current schedule
    fetch(`/api/medico/${doctorUserId}/horarios`)
      .then(r => r.json())
      .then(setSchedule);
  }, [doctorUserId]);
  
  const toggleDay = (dia_semana) => {
    setSchedule(schedule.map(day => 
      day.dia_semana === dia_semana 
        ? { ...day, activo: !day.activo }
        : day
    ));
  };
  
  const updateTime = (dia_semana, field, value) => {
    setSchedule(schedule.map(day =>
      day.dia_semana === dia_semana
        ? { ...day, [field]: value }
        : day
    ));
  };
  
  const saveSchedule = async () => {
    // Only send active days
    const activeSchedule = schedule.filter(day => day.activo);
    
    const result = await updateDoctorSchedule(doctorUserId, activeSchedule);
    alert(result.message);
  };
  
  return (
    <div>
      {schedule.map(day => (
        <DayEditor
          key={day.dia_semana}
          day={day}
          onToggle={toggleDay}
          onTimeChange={updateTime}
        />
      ))}
      <button onClick={saveSchedule}>Save Schedule</button>
    </div>
  );
};
```

## Validation Tips

```javascript
const validateSchedule = (schedule) => {
  const errors = [];
  
  schedule.forEach(day => {
    if (!day.activo) return; // Skip inactive days
    
    // Validate time format
    if (!/^\d{2}:\d{2}$/.test(day.hora_inicio)) {
      errors.push(`Invalid start time format for day ${day.dia_semana}`);
    }
    
    // Validate end time is after start time
    const [startH, startM] = day.hora_inicio.split(':').map(Number);
    const [endH, endM] = day.hora_fin.split(':').map(Number);
    const startMinutes = startH * 60 + startM;
    const endMinutes = endH * 60 + endM;
    
    if (endMinutes <= startMinutes) {
      errors.push(`End time must be after start time for day ${day.dia_semana}`);
    }
    
    // Validate appointment duration
    if (day.duracion_cita < 15 || day.duracion_cita > 240) {
      errors.push(`Appointment duration should be between 15-240 minutes`);
    }
    
    // Validate duration fits in working hours
    const totalMinutes = endMinutes - startMinutes;
    if (totalMinutes < day.duracion_cita) {
      errors.push(`Working hours too short for appointment duration on day ${day.dia_semana}`);
    }
  });
  
  return errors;
};
```

## Notes

- Requires authentication (doctor must be logged in)
- Uses database transactions - if any part fails, no changes are saved
- **Replaces entire schedule** - always include all working days in your request
- Inactive days don't need to be included in the request
- Time format must be HH:MM in 24-hour format
- Changes take effect immediately for new appointment bookings
- Existing booked appointments are not affected by schedule changes
- Appointment duration can be any value in minutes, but typically: 30, 45, or 60
