## Request Body

<ParamField body="NOMBRE_COMPLETO" type="string" required>
  Full name of the patient (max 255 characters)
</ParamField>

<ParamField body="CORREO" type="string" required>
  Patient's email address (must be valid email format, max 100 characters)
</ParamField>

<ParamField body="SEXO" type="string" optional>
  Patient's sex (single character: 'F' for female or 'M' for male)
</ParamField>

<ParamField body="TELEFONO" type="string" optional>
  Contact phone number (numeric)
</ParamField>

<ParamField body="ALERGIAS_PRINCIPALES" type="string" optional>
  Main allergies or medical conditions the patient has
</ParamField>

<ParamField body="TIPO_SANGRE" type="string" optional>
  Blood type (e.g., 'O+', 'A-', 'B+', 'AB-', max 10 characters)
</ParamField>

<ParamField body="ID_USUARIO" type="number" optional>
  Associated user account ID to link this patient record to an existing user
</ParamField>

## Implementation Details

### Oracle Database

This endpoint inserts into the `PACIENTES` table:

- **Table**: `PACIENTES`
- **Primary Key**: `ID_PACIENTE` (auto-generated by sequence `SEQ_PACIENTES`)
- **Sequence**: The Oracle sequence automatically generates the next ID
- **Timestamps**: Disabled - no created_at/updated_at columns

### Controller Method

Implemented in `PacienteController::store()` at `app/Http/Controllers/PacienteController.php:24`

```php
public function store(Request $request)
{
    // 1. Validation
    $request->validate([
        'NOMBRE_COMPLETO' => 'required|string|max:255',
        'CORREO' => 'required|email|max:100',
    ]);

    // 2. Create object
    $paciente = new Paciente();
    $paciente->NOMBRE_COMPLETO = $request->NOMBRE_COMPLETO;
    $paciente->CORREO = $request->CORREO;
    $paciente->SEXO = $request->SEXO;
    $paciente->TELEFONO = $request->TELEFONO;
    $paciente->ALERGIAS_PRINCIPALES = $request->ALERGIAS_PRINCIPALES;
    $paciente->TIPO_SANGRE = $request->TIPO_SANGRE;

    if ($request->has('ID_USUARIO')) {
        $paciente->ID_USUARIO = $request->ID_USUARIO;
    }

    // 3. Save to Oracle
    $paciente->save();

    return response()->json([
        'message' => 'Paciente registrado correctamente',
        'paciente' => $paciente
    ], 201);
}
```

## Validation Rules

<Warning>
  **Required Fields**
  - `NOMBRE_COMPLETO`: Required, string, maximum 255 characters
  - `CORREO`: Required, valid email format, maximum 100 characters
</Warning>

<Info>
  All other fields are optional and will be stored as provided. The `ID_USUARIO` field allows linking this patient record to an existing user account in the USUARIOS table.
</Info>

## Notes

<Note>
  **Oracle Sequence**: The `ID_PACIENTE` is automatically generated by the Oracle sequence `SEQ_PACIENTES`. You do not need to provide this value in the request.
</Note>

<Tip>
  When creating a patient record for a logged-in user, pass the user's `ID_USUARIO` to establish the relationship between the user account and medical records.
</Tip>
