  POST

## Authentication

This endpoint is currently **public** and does not require authentication.

<Warning>
  In production, this endpoint MUST be protected with authentication to prevent unauthorized profile modifications.
</Warning>

## Request Body

This endpoint accepts `multipart/form-data` to support file uploads.

<ParamField body="ID_USUARIO" type="number" required>
  The user ID from USUARIOS table (must exist in the database)
</ParamField>

<ParamField body="NOMBRE_COMPLETO" type="string" required>
  Full name of the patient (max 255 characters)
</ParamField>

<ParamField body="TELEFONO" type="string" optional>
  Contact phone number (numeric values only)
</ParamField>

<ParamField body="FECHA_NACIMIENTO" type="string" optional>
  Date of birth in YYYY-MM-DD format
</ParamField>

<ParamField body="TIPO_SANGRE" type="string" optional>
  Blood type (max 10 characters, e.g., 'O+', 'A-', 'B+', 'AB-')
</ParamField>

<ParamField body="ALERGIAS_PRINCIPALES" type="string" optional>
  Main allergies or medical conditions
</ParamField>

<ParamField body="SEXO" type="string" optional>
  Patient's sex (single character: 'F' or 'M')
</ParamField>

<ParamField body="FOTO_PERFIL" type="file" optional>
  Profile photo image file (max 2MB, image formats only: jpg, png, gif, etc.)
</ParamField>

## Implementation Details

### Controller Method

Implemented in `PacienteController::updateProfile()` at `app/Http/Controllers/PacienteController.php:57`

```php
public function updateProfile(Request $request)
{
    // 1. Validate (Medical + Photo)
    $request->validate([
        'ID_USUARIO' => 'required|exists:USUARIOS,ID_USUARIO',
        'NOMBRE_COMPLETO' => 'required|string|max:255',
        'TELEFONO' => 'nullable|numeric',
        'FOTO_PERFIL' => 'nullable|image|max:2048',
        'FECHA_NACIMIENTO' => 'nullable|date',
        'TIPO_SANGRE' => 'nullable|string|max:10',
        'ALERGIAS_PRINCIPALES' => 'nullable|string',
        'SEXO' => 'nullable|string|max:1',
    ]);

    // 2. Handle photo upload
    $rutaFoto = null;
    if ($request->hasFile('FOTO_PERFIL')) {
        // Save to storage/app/public/avatars
        $rutaFoto = $request->file('FOTO_PERFIL')->store('avatars', 'public');
        
        // Update USUARIOS table with photo path
        User::where('ID_USUARIO', $request->ID_USUARIO)
            ->update(['FOTO_PERFIL' => $rutaFoto]);
    }

    // 3. Create or update patient record
    $paciente = Paciente::where('ID_USUARIO', $request->ID_USUARIO)->first();

    if (!$paciente) {
        $paciente = new Paciente();
        $paciente->ID_USUARIO = $request->ID_USUARIO;
        $paciente->CORREO = $user->CORREO ?? 'sin_correo@vita.fem';
    }

    // Update medical data
    $paciente->NOMBRE_COMPLETO = $request->NOMBRE_COMPLETO;
    $paciente->TELEFONO = $request->TELEFONO;
    $paciente->FECHA_NACIMIENTO = $request->FECHA_NACIMIENTO;
    $paciente->TIPO_SANGRE = $request->TIPO_SANGRE;
    $paciente->ALERGIAS_PRINCIPALES = $request->ALERGIAS_PRINCIPALES;
    $paciente->SEXO = $request->SEXO;

    $paciente->save();

    return response()->json([
        'message' => 'Perfil actualizado correctamente',
        'paciente' => $paciente,
        'foto_url' => $rutaFoto ? asset('storage/' . $rutaFoto) : null
    ]);
}
```

## Behavior

### Update or Create (Upsert)

This endpoint implements an "upsert" pattern:

1. **If patient record exists** (matching `ID_USUARIO`): Updates existing record
2. **If patient record doesn't exist**: Creates new record with provided data

<Info>
  This allows users to complete their profile information even if they didn't fill it out during registration.
</Info>

### Photo Upload Process

1. File is validated (must be image, max 2MB)
2. File is stored in `storage/app/public/avatars/` with a random name
3. Relative path (e.g., `avatars/abc123.jpg`) is saved to `USUARIOS.FOTO_PERFIL`
4. Full URL is returned in the response using `asset()` helper

<Warning>
  **Storage Setup**: Ensure the symbolic link exists between `public/storage` and `storage/app/public` by running:
  ```bash
  php artisan storage:link
  ```
</Warning>

## Error Handling

The endpoint includes comprehensive error handling:

- **Validation errors** (422): Returns detailed field-level errors
- **Server errors** (500): Logs error and returns error message
- All errors are logged using Laravel's `Log::error()` for debugging

## Database Tables Affected

1. **PACIENTES**: Medical and personal information (created or updated)
2. **USUARIOS**: Profile photo path (updated only if photo uploaded)

<Tip>
  The endpoint updates two separate tables to maintain a clean separation between authentication data (USUARIOS) and medical records (PACIENTES).
</Tip>
