## Overview

This endpoint returns all appointments for a specific patient, including completed consultations with medical records, diagnoses, and prescriptions. Results are ordered from most recent to oldest.

<Info>
  This is a **public endpoint** but should be protected on the client side to ensure patients only access their own appointments using their `id_usuario`.
</Info>

## Path Parameters

<ParamField path="id_usuario" type="integer" required>
  Patient's user ID from the `USUARIOS` table. This is the same as `ID_PACIENTE` in the appointments.
</ParamField>

## Response Fields

<ResponseField name="id_cita" type="integer" required>
  Unique appointment identifier
</ResponseField>

<ResponseField name="fecha_formateada" type="string" required>
  Formatted appointment date in `dd/mm/yyyy` format (e.g., "15/03/2026")
</ResponseField>

<ResponseField name="hora" type="string" required>
  Appointment time in 24-hour `HH:mm` format (e.g., "14:30")
</ResponseField>

<ResponseField name="medico" type="string" required>
  Doctor's full name. Falls back to "Especialista" if doctor information is unavailable.
</ResponseField>

<ResponseField name="estado" type="string" required>
  Current appointment status:
  - `PENDIENTE` - Awaiting confirmation
  - `CONFIRMADA` - Confirmed by administrative staff
  - `CANCELADA` - Cancelled by patient or staff
  - `FINALIZADA` - Completed with medical consultation recorded
</ResponseField>

<ResponseField name="motivo" type="string">
  Reason for the appointment as provided during booking
</ResponseField>

<ResponseField name="consulta_medica" type="object">
  Medical consultation details. Only present for `FINALIZADA` appointments where the doctor has recorded the consultation.
</ResponseField>

<ResponseField name="consulta_medica.diagnostico" type="string">
  Doctor's diagnosis from the consultation
</ResponseField>

<ResponseField name="consulta_medica.indicaciones" type="string">
  Treatment instructions and recommendations
</ResponseField>

<ResponseField name="consulta_medica.medicamentos" type="array">
  Prescribed medications with dosage and duration
</ResponseField>

<ResponseField name="consulta_medica.medicamentos[].medicamento" type="string">
  Medication name
</ResponseField>

<ResponseField name="consulta_medica.medicamentos[].dosis" type="string">
  Dosage amount (e.g., "500mg", "2 tabletas")
</ResponseField>

<ResponseField name="consulta_medica.medicamentos[].frecuencia" type="string">
  Frequency of administration (e.g., "Cada 8 horas", "2 veces al día")
</ResponseField>

<ResponseField name="consulta_medica.medicamentos[].duracion" type="string">
  Treatment duration (e.g., "7 días", "2 semanas")
</ResponseField>

## Implementation Details

### Data Retrieval

The endpoint uses Eloquent eager loading to optimize queries:

```php
Cita::with(['medico.usuario'])
    ->where('ID_PACIENTE', $id_usuario)
    ->orderBy('FECHA_HORA', 'desc')
    ->get();
```

### Medical Consultation Loading

For each appointment, the system:
1. Checks if a `CONSULTA` record exists for the appointment
2. If found, retrieves the associated `RECETA` (prescription)
3. Loads all `DETALLE_RECETA` records (individual medications)
4. Assembles the complete medical record

```php
$consulta = Consulta::where('ID_CITAS', $idCitaReal)->first();
if ($consulta) {
    $receta = Receta::where('ID_CONSULTA', $idConsultaReal)->first();
    if ($receta) {
        $medicamentos = DetalleReceta::where('ID_RECETA', $idRecetaReal)->get();
    }
}
```

### Oracle Compatibility

The controller handles Oracle's case sensitivity throughout:

```php
$fechaHoraStr = $cita->FECHA_HORA ?? $cita->fecha_hora;
$diagnostico = $consulta->DIAGNOSTICO ?? $consulta->diagnostico;
```

## Appointment States Explained

<Steps>
  <Step title="PENDIENTE">
    Initial state when appointment is created. Awaiting administrative confirmation.
  </Step>
  
  <Step title="CONFIRMADA">
    Staff has verified the appointment. Patient should attend at the scheduled time.
  </Step>
  
  <Step title="FINALIZADA">
    Doctor has completed the consultation and recorded medical notes. `consulta_medica` object will be populated.
  </Step>
  
  <Step title="CANCELADA">
    Appointment was cancelled. Time slot becomes available for other patients.
  </Step>
</Steps>

## Data Privacy Considerations

<Warning>
  This endpoint returns sensitive medical information including diagnoses and prescriptions. Implement proper authentication and ensure patients can only access their own records.
</Warning>

<Check>
  **Recommended Security:**
  - Verify the `id_usuario` matches the authenticated user's ID
  - Implement rate limiting to prevent data scraping
  - Use HTTPS for all API calls
  - Log access to medical records for compliance
</Check>
