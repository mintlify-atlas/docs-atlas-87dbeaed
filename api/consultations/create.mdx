## Overview

The `atenderCita` endpoint allows physicians to complete an appointment by creating a comprehensive medical consultation record. This endpoint creates entries in three interconnected tables:

- **CONSULTAS**: Stores vital signs, symptoms, physical examination, diagnosis, and treatment
- **RECETAS**: Creates a prescription header linked to the consultation
- **DETALLE_RECETAS**: Stores individual medication details for the prescription

After successful completion, the appointment status is updated to `FINALIZADA`.

## Endpoint

```
PUT /api/medico/citas/{id}/atender
```

## Database Operations

This endpoint performs the following operations in a database transaction:

1. **Create Consultation Record**: Inserts into `CONSULTAS` table with:
   - Link to appointment (ID_CITAS)
   - Patient ID (ID_PACIENTE) from the appointment
   - All vital signs and clinical data

2. **Create Prescription (if medications provided)**:
   - Inserts into `RECETAS` table with current timestamp
   - Links to consultation via ID_CONSULTA

3. **Create Medication Details**:
   - Inserts each medication into `DETALLE_RECETAS`
   - Links to prescription via ID_RECETA

4. **Update Appointment**:
   - Sets `ESTADO` to `'FINALIZADA'`
   - Stores diagnosis summary in `NOTAS_MEDICAS`

If any operation fails, the entire transaction is rolled back.

## Notes

- The endpoint uses Oracle database with case-sensitive column names (uppercase)
- All fields except `medicamentos` array are required
- Prescriptions are optional - omit the `medicamentos` array if no medications are prescribed
- The appointment must exist and belong to a valid patient
- Returns 404 if the appointment ID is not found
