<ParamField path="method" type="string" required>
  POST
</ParamField>

## Authentication

This endpoint is public but requires a valid Firebase ID token in the request body.

## Request Body

<ParamField body="idToken" type="string" required>
  Firebase ID token obtained from Firebase Authentication SDK on the client side
</ParamField>

## Response

<ResponseField name="token" type="string">
  Sanctum authentication token for subsequent API requests
</ResponseField>

<ResponseField name="user" type="object">
  The authenticated or newly created user object
  
  <ResponseField name="id" type="integer">
    Unique user identifier
  </ResponseField>
  
  <ResponseField name="name" type="string">
    User's full name from Firebase
  </ResponseField>
  
  <ResponseField name="email" type="string">
    User's email address from Firebase
  </ResponseField>
  
  <ResponseField name="firebase_uid" type="string">
    Firebase user identifier
  </ResponseField>
  
  <ResponseField name="created_at" type="string">
    Account creation timestamp
  </ResponseField>
  
  <ResponseField name="updated_at" type="string">
    Last update timestamp
  </ResponseField>
</ResponseField>

## Implementation Flow

### Step 1: Authenticate with Firebase (Client Side)

```javascript Firebase Client Authentication
import { getAuth, signInWithPopup, GoogleAuthProvider } from 'firebase/auth';

const auth = getAuth();
const provider = new GoogleAuthProvider();

try {
  const result = await signInWithPopup(auth, provider);
  const user = result.user;
  
  // Get the Firebase ID token
  const idToken = await user.getIdToken();
  
  // Send to your backend
  const response = await fetch('https://api.vitafem.com/api/auth/firebase', {
    method: 'POST',
    headers: {
      'Content-Type': 'application/json',
      'Accept': 'application/json'
    },
    body: JSON.stringify({ idToken })
  });
  
  const data = await response.json();
  
  // Store the Sanctum token
  localStorage.setItem('authToken', data.token);
  
} catch (error) {
  console.error('Authentication error:', error);
}
```

### Step 2: Use Sanctum Token for API Requests

```javascript Authenticated API Request
const token = localStorage.getItem('authToken');

fetch('https://api.vitafem.com/api/me', {
  headers: {
    'Authorization': `Bearer ${token}`,
    'Accept': 'application/json'
  }
})
.then(response => response.json())
.then(data => console.log(data));
```

## How It Works

1. **Client authenticates with Firebase**: User signs in using Firebase Authentication SDK (Google, Email/Password, etc.)
2. **Client obtains ID token**: Firebase returns an ID token after successful authentication
3. **Client sends token to backend**: ID token is sent to this endpoint
4. **Backend verifies token**: Laravel backend uses Firebase Admin SDK to verify the token authenticity
5. **Backend extracts user info**: Retrieves email, name, and Firebase UID from verified token
6. **User lookup/creation**:
   - Searches for user by `firebase_uid` or `email`
   - Creates new user if not found
   - Links `firebase_uid` to existing user if found by email
7. **Token generation**: Creates Sanctum authentication token
8. **Response**: Returns Sanctum token and user data

## User Management Logic

### New User Creation

If no user exists with the Firebase UID or email:
- Creates new user account
- Sets name from Firebase (or email if name unavailable)
- Generates random secure password
- Stores Firebase UID

### Existing User Linking

If user exists by email but has no Firebase UID:
- Links Firebase UID to existing account
- Updates user record

If user exists by Firebase UID:
- Returns existing user
- No modifications needed

## Security Features

- **Token Verification**: Firebase Admin SDK verifies token signature and expiration
- **Secure Passwords**: Random 24-character passwords generated for Firebase users
- **UID Linking**: Firebase UID prevents duplicate accounts
- **Error Logging**: Server errors logged for monitoring and debugging
- **Email Required**: Authentication fails if Firebase doesn't provide email

## Notes

- Requires Firebase Admin SDK configuration in Laravel backend
- ID tokens expire after 1 hour (standard Firebase behavior)
- Email must be available from Firebase authentication method
- Supports all Firebase authentication methods (Google, Email/Password, Phone, etc.)
- The returned Sanctum token should be used for all subsequent API requests
- Firebase UID is stored for account linking and identification
