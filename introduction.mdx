## Overview

VitaFem Backend API is a robust Laravel-based healthcare management system designed to handle patient information, medical appointments, veterinary services, and administrative operations. Built with Laravel 8.75 and Oracle database integration, it provides a complete backend solution for healthcare applications.

## Technology Stack

### Core Framework
- **Laravel 8.75** - PHP web application framework
- **PHP 8.2+** - Latest PHP version support
- **Oracle Database** - Enterprise-grade database via yajra/laravel-oci8

### Authentication & Security
- **Laravel Sanctum** - API token authentication
- **Firebase PHP SDK** (kreait/firebase-php) - Firebase authentication
- **Laravel Socialite** - OAuth social login (Google, Facebook)
- **Firebase JWT** - JSON Web Token handling

## API Architecture

### Base URL
```
http://localhost:8000/api
```

### Authentication Methods

The API supports three authentication methods:

1. **Sanctum Token Authentication** - For traditional email/password login
2. **Firebase Authentication** - For Firebase-authenticated users
3. **Social OAuth** - For Google and Facebook login

### Route Protection

Routes are organized into two categories:

<CodeGroup>
```php Public Routes
// No authentication required
POST /api/login
POST /api/register
GET  /api/productos
POST /api/citas
GET  /api/users
POST /api/chatbot/message
```

```php Protected Routes
// Requires auth:sanctum middleware
POST /api/logout
GET  /api/me
GET  /api/citas
GET  /api/pets
POST /api/appointments
PUT  /api/users/{id}
DELETE /api/citas/{id}
```
</CodeGroup>

## Database Schema

The API uses an Oracle database with the following key tables:

- **USUARIOS** - User accounts and authentication
- **CITAS** - Medical appointments
- **PACIENTES** - Patient profiles
- **PRODUCTOS** - Healthcare products
- **Pets** - Veterinary patient records

<Note>
All Oracle table and column names are in UPPERCASE. The API handles mapping between Laravel conventions (lowercase) and Oracle naming (UPPERCASE).
</Note>

## Response Format

All API responses follow a consistent JSON format:

```json Success Response
{
  "message": "Operation successful",
  "data": { },
  "token": "string" // when applicable
}
```

```json Error Response
{
  "message": "Error description",
  "errors": { } // validation errors when applicable
}
```

## Rate Limiting

The API uses Laravel's default rate limiting:
- **60 requests per minute** for authenticated routes
- Throttled by user token

## CORS Configuration

CORS is configured via `fruitcake/laravel-cors` package:
- Supports cross-origin requests
- Configured for frontend integration
